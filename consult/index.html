Phased Project Approach

Through my years as a consultant I have honed a project approach that is carried out in distinct, linear phases. These phases call out specific milestones that are marked by deliverables. Deliverables can take the form of documents, functional systems, or accomplishments. At the conclusion of each project phase, success will be measured and evaluated by each party. Sometimes, the conclusion of a phase is the appropriate time to adjust the project deliverables, or even its goals and scope.

Phase 1: Discover

The first phase of solution delivery, this initiation phase is critical in establishing baseline expectations, environmental details, operational challenges and determining project success criteria. This phase includes network and environmental discovery, wherein many of the assumptions made during the proposal process may be validated, representing an opportunity to mitigate concerns early before the project progresses any further.

Phase 2: Design and Plan

Following the discovery phase, the design and planning phase serves to map out the remainder of the project and further develop the high-level design presented during the proposal process. The core project documentation is drafted, including Project Plan, Design, Implementation Plan, and Test Documents. Collaborative design sessions allow our team to work closely with your IT operations and management staff, facilitating a reciprocal exchange as all design aspects are thoroughly discussed and determined.

Phase 3: Test

Build the lab environment and test it against Test and Implementation Procedures. Build the pilot environment and test it against Implementation Procedures, User Experience, and Success Criteria. Revise Project Plan and Documentation against lessons learned.

Phase 4: Implement

Measure three times, cut once. You might notice that it seems like most of the work is done before the Implementation Phase. That is true. I believe you will be more efficient in the long run by being deliberate and careful in planning and testing before you take action.

Phase 5: Wrap Up

The environment is supported as needed. Documentation is revised as necessary. Final Knowledge Transfer is performed. Project debrief is held with lessons learned and outstanding issues recorded.
